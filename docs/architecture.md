## Компоненты

### Crypto (Rust)
- Генерация ключей Ed25519.
- Выполнение Noise handshake.
- Симметричное шифрование (AES-256-GCM).
- Подписи и верификация.

### P2P (Rust, libp2p)
- Создание узла libp2p (идентификатор на основе публичного ключа).
- Поведения (behaviours): Kademlia, mDNS, Ping, Identify, Relay.
- Hole punching (DCUtR).
- Отправка/приём зашифрованных кадров через установленные соединения.

### Storage (Rust)
- Локальное хранение контактов, ключей, истории сообщений.
- Используется SQLite или просто файлы (в зависимости от платформы).
- Потокобезопасный доступ.

### GUI (Flutter)
- MVVM-подобная архитектура (Bloc/Cubit).
- Визуализация чатов, контактов, настроек.
- Вызов Rust-функций через FFI.

## Поток данных (сообщение от А к Б онлайн)

1. Пользователь А вводит номер телефона Б → UI вызывает Rust-функцию `lookup_contact(phone_hash)`.
2. Бэкенд ищет в DHT запись, получает публичный ключ Б.
3. Бэкенд ищет актуальный мультиадрес Б (также через DHT).
4. Устанавливается Noise-соединение (hole punching).
5. А отправляет кадр с типом `0x01` (текст).
6. Б получает, дешифрует, отправляет Ack.
7. Б сохраняет сообщение в локальное хранилище, UI уведомляется через событие.

## Офлайн-сценарий

- Если шаг 3 не дал мультиадреса (Б офлайн), отправитель:
  - Генерирует эфемерный ключ.
  - Шифрует сообщение по публичному ключу Б.
  - Помещает в DHT по ключу `/offline/ключ_Б`.
- При появлении Б онлайн он сканирует DHT и забирает сообщения.
