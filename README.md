# Ring-Ring

Ring‑Ring — это некоммерческий, учебный проект по созданию P2P-мессенджера с открытым исходным кодом. Проект объединяет архитектурные принципы классического Skype (децентрализация, P2P, устойчивость) и открытость кода Telegram (лицензия GNU GPLv3).

**Основные принципы:**
*   **Privacy by design**: Никакой сервер не хранит переписки и ключи пользователей.
*   **Полная децентрализация (P2P)**: Участники общаются напрямую.
*   **Сквозное шифрование**: Вся переписка шифруется на устройстве отправителя.
*   **Открытый код и модульность**: Каждый может изучить, доработать или использовать протокол в своих проектах.
*   **Образовательная цель**: Изучение современных сетевых и криптографических технологий через практическую реализацию.

**Ключевые технологии:**
*   **Язык**: Rust (библиотеки и бэкенд) для производительности и надёжности, Flutter (фронтенд) для кроссплатформенного UI.
*   **Транспорт и сеть**: libp2p (Kademlia DHT, mDNS, hole punching).
*   **Криптография**: Шифрование трафика — стандартизированный протокол Noise (libp2p Noise) с использованием кривых X25519 и AES‑256‑GCM. Для первичного обмена ключами (в рамках протокола регистрации) допустимо использование асимметричной криптосистемы RSA с длиной ключа не менее 2048 бит. Сессионное шифрование гарантирует Perfect Forward Secrecy.
*   **Аутентификация**: Локальная генерация пары ключей Ed25519; номер телефона используется только для первичного обнаружения контакта (DHT хранит хэш номера и публичный ключ).

**Регистрация и аутентификация:**
Пользователь вводит номер телефона в клиент, получает одноразовый код от независимого SMS-провайдера и вводит его в приложение. Клиент генерирует локальную пару ключей. В DHT публикуется запись: хэш(номер_телефона) → подписанный публичный ключ.
При поиске контакта пользователь вводит номер, клиент вычисляет его хэш, находит в DHT запись, получает публичный ключ и устанавливает с ним прямое зашифрованное соединение.
Настройки конфиденциальности позволяют скрыть номер телефона от всех, кроме избранных контактов (по умолчанию — «никто»).

**План реализации:**
1.  Разработка и спецификация протокола.
2.  Написание базовой библиотеки на Rust (генерация ключей, работа с DHT, Noise-соединения).
3.  CLI-клиент для тестирования базовых функций.
4.  GUI-клиент (Flutter) для iOS, Android и десктопов.
5.  Поддержка голосовых и видеозвонков.
6.  Групповые чаты.
7.  Всестороннее тестирование и аудит безопасности.
